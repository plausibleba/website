Canvas → VCC Handoff

The complete user experience from generating an operating model on Canvas through to landing inside VCC with it loaded as a project.

User journey
📝
plausibleba.com/canvas
User arrives at Canvas
They land on the Canvas intake page. They can paste meeting notes, upload a transcript (Word, PDF, Excel), or click "Load the PortfolioProp demo" to try it with sample data. They enter their name and email to pass the email gate.
⚡
plausibleba.com/canvas
AI generates the operating model
The LLM pipeline builds a BIZBOK-grounded operating model: capability map, concept model, value streams, and metrics. The model renders as an interactive preview with tabs for each element type. Users can explore, browse tabs.
Rate limit: 3 free generations per email. After 3, they see "You've used all 3 free generations. Open in VCC for unlimited access."
👉
plausibleba.com/canvas
User clicks "Open in VCC →"
Below the rendered model, a CTA banner reads: "Turn this into an interactive workshopping canvas." The user clicks "Open in VCC →". The button shows "Preparing…" for about a second.
Behind the scenes: the full bundle JSON is POSTed to /api/claim-bundle along with the user's email, first and last name (captured during the email gate). The API stores it in Vercel KV with a 24-hour TTL and returns a claim token like bndl_a8f3x9k2m7pq.
🚀
Redirect
Redirect to VCC with claim token
The browser redirects to VCC with the claim token and pre-fill data in the URL.
URL: https://app.plausibleba.com/?claim=bndl_xxx&email=...&firstName=...&lastName=...

On arrival, VCC's extractClaimFromURL() runs immediately: stashes the token in sessionStorage (so it survives the auth redirect), saves the pre-fill data, and cleans the URL via history.replaceState.
🔒
VCC App
Authentication
What happens next depends on whether the user already has a VCC session.
Path A — New user

Login page with context

They see the VCC login page, but it's customised for the handoff:

• Green banner: "Your operating model is ready to import"
• Subtitle: "Sign in to open your operating model"
• Email field pre-filled from the Canvas session
• Button reads "Sign in & import model"

They sign in with Google or request a magic link. After auth completes, the flow continues automatically.

Path B — Existing session

Skips login entirely

If the user is already signed in to VCC (e.g. in another tab, or a persistent session), the login page never appears. The claim consumer fires immediately after auth state is confirmed.

📦
VCC App
Bundle auto-imports
A spinner screen appears: "Importing your operating model… This will only take a moment." Behind the scenes:
1. Read claim token from sessionStorage
2. Fetch bundle from plausibleba.com/api/claim-bundle?token=bndl_xxx
3. Token is consumed (one-time use — deleted on retrieval)
4. Bundle is normalised (PlausibleBA format → VCC internal scaffold)
5. Scaffold loaded into the canvas store
6. autoSaveToProject() creates a Supabase project automatically
✔️
VCC App
User lands on Network view
The spinner clears and the user is looking at their operating model inside VCC — the Network view showing value streams, capabilities, and concepts. A project has been created automatically in their account. They can now navigate to Stream, Capabilities, Concepts, Friction, and start workshopping.
What the user never had to do: download a JSON file, find an import button, navigate to the right screen, or re-enter any information. The bundle survived the entire auth flow without them touching it.
Edge cases & failure modes
Scenario Handling What the user sees
Token expired (24h+) Silent Normal login/project list. No error. They can go back to Canvas and click "Open in VCC" again, or use Import Bundle.
Token already claimed Silent Same as expired — claim API returns 404, consumer clears state, user sees normal VCC.
Claim API unreachable Graceful Console logs error. User lands on normal project list. No visible error — they can still import manually.
Bookmarked claim URL Silent Token consumed on first visit. Bookmark loads normal VCC with no claim params.
Unrecognised bundle format Graceful Console warning. Import skipped silently. User sees project list.
User clicks "Open in VCC" with no bundle Fallback Alert: "No bundle to open. Generate or upload a model first."
Claim API POST fails (Canvas side) Fallback Alert: "Could not connect to VCC. You can download the bundle and import it manually." Button resets.
Local dev mode (no Supabase) Silent Claim consumer skipped entirely (isLocalMode guard). Normal local VCC experience.
User closes tab mid-auth Graceful Token stays in sessionStorage (tab-scoped). If they re-open VCC from the magic link email, sessionStorage is empty — normal login. They can re-click "Open in VCC" from Canvas.
