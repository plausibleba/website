How it works

From description to operating model in three steps.

No specialist tooling. No training. No install. Just paste what you know about the business.

What you get

Three artefacts. One conversation.

Each artefact cross-validates the others. Every business object should ground at least one capability. Every capability should appear in at least one value stream stage.

Capability Map

What the organisation can do

A MECE capability hierarchy grounded in BIZBOK. L1–L3 with Execution and Governance domains distinguished. Validated at every level.

Output: Interactive treemap + XLSX workbook
Concept Model

What the organisation manages

Business objects classified as Party, Record, or Resource using the Capsicum Triad. Lifecycle states per object. Cross-validated against the capability map.

Output: Interactive concept graph + XLSX workbook
Value Stream

How the organisation delivers value

Staged value delivery with outcome chain, capability-to-stage mapping, and PPIT breakdown. Exports ba-skills-bundle.json for VCC pipeline import.

Output: Interactive stage view + XLSX + JSON bundle
The methodology

BIZBOK-grounded. Not just formatted.

Without a methodology, asking any AI to draw a capability map produces a list. PlausibleBA embeds structured elicitation, MECE validation, and schema-validated outputs — because the methodology is baked in, not bolted on.

BIZBOK-grounded

Capability naming conventions, Execution/Governance split, and value stream anatomy from the Business Architecture Guild — not invented on the fly.

Capsicum Triad classification

Business objects classified as Party, Record, or Resource — grounded in REA theory (McCarthy 1982) and Bunge's ontology. Every object must ground at least one capability.

VCC pipeline-ready

Every output exports a schema-validated ba-skills-bundle.json that imports directly into the Value Cognition Canvas for friction analysis and workshopping.
